## ğŸ“ Contribution Guidelines

### For Code Contributions

1. **Fork the Repository**
   ```bash
   git clone https://github.com/your-username/100-days-of-machine-learning.git
   cd 100-days-of-machine-learning
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-contribution
   ```

3. **Make Your Changes**
   - Follow the existing directory structure
   - Place notebooks in the appropriate section folder
   - Write clear, commented code
   - Use meaningful commit messages
   - Ensure notebooks run without errors

4. **Test Your Changes**
   - Ensure code runs without errors
   - Check that notebooks execute properly
   - Verify all links work
   - Test with different datasets if applicable

5. **Submit a Pull Request**
   - Provide a clear description of your changes
   - Reference any related issues
   - Include screenshots for visual changes
   - Be open to feedback and discussion

### Notebook Standards

- **Clear Structure**: Use markdown cells for explanations
- **Comments**: Add comments to explain complex logic
- **Visualizations**: Include plots and charts where appropriate
- **Examples**: Provide multiple examples when possible
- **Documentation**: Add docstrings for functions
- **Output**: Include expected outputs in markdown cells

### Code Style

- **Python**: Follow PEP 8 guidelines
- **Comments**: Write clear, helpful comments
- **Documentation**: Include docstrings for functions
- **Naming**: Use descriptive variable and function names
- **Formatting**: Use consistent formatting throughout
